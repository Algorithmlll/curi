@curi/mobx

v1.0.0-alpha.0
GitHub RepoNPM Package

About#

This package allows you to synchronize the response objects generated by Curi with MobX. Whenever location changes occur, the store will be updated.

Installation#

npm install @curi/mobx
yarn add @curi/mobx

If you prefer to use script tags, Unpkg will always have the latest version of @curi/mobx available for you.

<script src="https://unpkg.com/@curi/mobx@1.0.0-alpha.0/dist/curi-mobx.js" />

There is also a minimized version available if you change the filename to mobx.min.js. The package will be attached to the window as window.CuriMobX.

API#

import CuriStore from '@curi/mobx';

CuriStore#

CuriStore will store responses. When creating a store you just need to pass it your router and the store will automatically listen for new responses and update itself.

const router = curi(history, routes);
const curiStore = new CuriStore(router);

With React, you can use the <Provider> from mobx-react to make your store available to components. You would then use the inject higher-order component to inject the Curi props into components. For example, you would inject the router, response, and action props into your <CuriBase>.

const ReactiveBase = inject(({ curi }) => ({
  router: curi.router,
  response: curi.response,
  action: curi.action
}))(observer(CuriBase));
ReactDOM.render((
  <Provider curi={curiStore}>
    <ReactiveBase render={...}/>
  </Provider>
), holder);
