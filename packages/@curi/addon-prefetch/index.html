@curi/addon-prefetch

v1.0.0-beta.6
GitHub RepoNPM Package

About#

The prefetch add-on can be used to make data fetching calls prior to navigation by calling a route's match.every function. This is different than calling the load function while generating the response because this is done without actually changing locations.

Installation#

npm install @curi/addon-prefetch
yarn add @curi/addon-prefetch

If you prefer to use script tags, Unpkg will always have the latest version of @curi/addon-prefetch available for you.

<script src="https://unpkg.com/@curi/addon-prefetch@1.0.0-beta.6/dist/curi-addon-prefetch.js" />

There is also a minimized version available if you change the filename to addon-prefetch.min.js. The package will be attached to the window as window.CuriAddonPrefetch.

Note: You should only use this if you implement some sort of caching/lookup in your load functions. The match.every function will be re-called when the user actually navigates to the route, so the benefit comes from the load function using a cached value instead of sending a new request to your server.

API#

createPrefetchAddon#

The default export function is an add-on factory that will add an prefetch function to your router object's add-on property.

import curi from '@curi/core';
import prefetch from '@curi/addon-prefetch';

const router = curi(history, routes, { addons: [prefetch()] });

The prefetch add-on allows you to call a route's load function manually. Why would you want to do this? Preloading data can give your users a faster navigation time when navigating to a page whose data has already been loaded.

Arguments#

// call a route's load function manually
router.addons.prefetch('User', { id: 2 })
  • name - the name of the route whose match.every function should be called.
  • props - route props that are used by the match.every function (the same ones that a match.every function expects).

This add-on will only register routes that have a match.every function.
