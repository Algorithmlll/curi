@curi/route-prefetch

v1.0.0-beta.7
GitHub RepoNPM Package

About#

The prefetch route interaction can be used to make data fetching calls prior to navigation by calling a route's on.every() function. This is different than calling the function while generating the response because this is done without actually changing locations.

Installation#

npm install @curi/route-prefetch
yarn add @curi/route-prefetch

Prefer inline scripts?

Note: You should only use this if you implement some sort of caching/lookup in your functions. The on.every() function will be re-called when the user actually navigates to the route, so the benefit comes from the using a cached value instead of sending a new request to your server.

API#

prefetch#

The default export function is a route interaction factory that will add an prefetch function to the router's route property.

import curi from '@curi/core';
import prefetch from '@curi/route-prefetch';

const router = curi(history, routes, {
  route: [prefetch()]
});

The prefetch route interaction allows you to call a route's on.every() function manually. Why would you want to do this? Prefetching data means that when users navigate, the new page will be full rendered faster because we already have the data.

Arguments#

// call a route's load function manually
router.route.prefetch('User', { params: { id: 2 }})
  • name - the name of the route whose on.every() function should be called.
  • props - route props that are used by the on.every() function (the same ones that a on.every() function expects).

This route interaction will only register routes that have a on.every() function.
