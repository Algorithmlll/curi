About#

The curi-addon-active package compares a location object's pathname to the current location in order to determine if the location object is "active". This can be restricted to complete matches or allow partial matches so that locations that represent an ancestor of the current location are also considered "active".

Installation#

You can install the addon-active package from NPM.

npm install @curi/addon-active

If you prefer to use script tags, Unpkg will always have the latest version of addon-active available for you.

<script src="https://unpkg.com/@curi/addon-active@1.0.0-beta.3/dist/curi-addon-active.js" />

There is also a minimized version available if you change the filename to addon-active.min.js. The package will be attached to the window as window.CuriAddonActive.

API#

createActiveAddon

curi-addon-active has one, default export function (so you can import it with whatever name you want to. It is an addon factory that will add an active function to your configuration object's addon property.

import createConfig from '@curi/core';
import createActiveAddon from '@curi/addon-active';

const config = createConfig(history, routes, {
  addons: [createActiveAddon]
});

The active addon function takes four arguments: the name of the route you want to check, the current response object, any params of the route that you want to check, and whether to consider partial matches as active. A partial match would occur when you check an ancestor route of the current route.

const isActive = config.addons.active('Some Route', response, { id: 10 });
