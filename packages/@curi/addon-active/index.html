@curi/addon-active

v1.0.0-beta.5
GitHub RepoNPM Package

About#

The @curi/addon-active package determines whether a route is "active" by comparing it to the current response. This can be restricted to complete matches or allow partial matches so that locations that represent an ancestor of the current location are also considered "active".

Installation#

npm install @curi/addon-active
yarn add @curi/addon-active

If you prefer to use script tags, Unpkg will always have the latest version of @curi/addon-active available for you.

<script src="https://unpkg.com/@curi/addon-active@1.0.0-beta.5/dist/curi-addon-active.js" />

There is also a minimized version available if you change the filename to addon-active.min.js. The package will be attached to the window as window.CuriAddonActive.

API#

createActiveAddon#

The default export, this function is an add-on factory that will add an active function to your router's add-on property.

The addon returns a boolean, true when a route is "active" (it matches the response's location) and false when it is not.

import curi from '@curi/core';
import createActiveAddon from '@curi/addon-active';

const router = curi(history, routes, {
  addons: [createActiveAddon()]
});

Arguments#

  • name - the name of the route to check.
  • response - the response to check the route against.
  • params - any route params for the route that is being checked
  • partial - when true, ancestor routes can be considered active. (default false)
const isActive = router.addons.active(
  'Some Route',
  response,
  { id: 10 },
  false
);
