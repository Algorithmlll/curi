@curi/svelte

v1.0.0-alpha.3
GitHub RepoNPM Package

About#

This package enables you to use Curi alongside Svelte. This is more of a proof of concept than a fleshed out routing solution and only provides bare routing functionality.

Installation#

You can install the svelte package from NPM.

npm install @curi/svelte

If you prefer to use script tags, Unpkg will always have the latest version of svelte available for you.

<script src="https://unpkg.com/@curi/svelte@1.0.0-alpha.3/dist/curi-svelte.js" />

There is also a minimized version available if you change the filename to svelte.min.js. The package will be attached to the window as window.CuriSvelte.

API#

setConfig

In order for the components provided by this package to work, they need to have access to your Curi config object.

import { setConfig } from '@curi/svelte';

const config = createConfig(history, routes);
setConfig(config);

getConfig

getConfig complements setConfig by returning the configuration object set by calling setConfig. This can be used to access the configuration object throughout your application. It is used internally by @curi/svelte components.

import { getConfig } from '@curi/svelte';

// config will be undefined if you haven't called setConfig yet!
const config = getConfig();

Usage

The following is one way to setup rendering for a Curi + Vue application.

import { setConfig } from '@curi/svelte';
const config = createConfig(history, routes);
setConfig(config);

const root = document.getElementById('root');
let view;
function subscriber(response) {
  if (view) {
    view.destroy();
  }
  view = new response.body({
    target: root,
    data: response
  });
}
config.subscribe(subscriber);
