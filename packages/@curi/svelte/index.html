@curi/svelte

v1.0.0-beta.1

About#

This package enables you to use Curi alongside Svelte. This package relies on the Svelte store.

Installation#

You can install the svelte package from NPM.

npm install @curi/svelte

If you prefer to use script tags, Unpkg will always have the latest version of svelte available for you.

<script src="https://unpkg.com/@curi/svelte@1.0.0-beta.1/dist/curi-svelte.js" />

There is also a minimized version available if you change the filename to svelte.min.js. The package will be attached to the window as window.CuriSvelte.

API#

<Link>#

In order for the components provided by this package to work, they need to have access to your Curi config object.

<div>
  <Link to='Home'>Home</Link>
  <Link to='User' params='{{ { userID: 5 } }}'>Profile</Link>
</div>
<script>
  import { Link } from '@curi/svelte';
  export default {
    components: { Link }
  }
</script>

Usage#

The components exported by @curi/svelte rely on Svelte's store. The store should include a curi property that has a config property. Future releases might also include response and action properties, but for now config is the only required property.

import { Store } from 'svelte/store';

const config = createConfig(history, routes);
const store = new Store({
  curi: { config }
});

As far as rendering your application goes, you should use the subscribe method provided by the Curi configuration object to re-render whenever a new response is emitted.

const config = createConfig(history, routes);

const store = new Store({
  curi: { config }
});

const root = document.getElementById('root');
let view;

config.subscribe(response => {
  if (view) {
    view.destroy();
  }
  view = new response.body({
    target: root,
    store,
    data: response
  });
});
