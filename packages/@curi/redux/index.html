@curi/redux

v1.0.0-beta.2
GitHub RepoNPM Package

About#

This package allows you to synchronize the response objects generated by Curi with your Redux store. Whenever location changes occur, they will be dispatched to your store. If you use this (with a React application), you do not have to use the <CuriBase>.

Installation#

npm install @curi/redux
yarn add @curi/redux

If you prefer to use script tags, Unpkg will always have the latest version of @curi/redux available for you.

<script src="https://unpkg.com/@curi/redux@1.0.0-beta.2/dist/curi-redux.js" />

There is also a minimized version available if you change the filename to redux.min.js. The package will be attached to the window as window.CuriRedux.

API#

import { syncResponses, curiReducer } from '@curi/redux';

syncResponses#

syncResponses is responsible for linking your Redux store with your Curi router. It subscribes to location changes emitted from your router with a function that will dispatch a "location changed" event to the Redux store. It will also add your Curi router to the store.

const router = curi(history, routes);
const store = createStore(reducer);

syncResponses(store, router);
// whenever navigation happens, the router will now
// pass the response and action to your Redux store.

curiReducer#

The curiReducer keeps track of your Curi router, the latest response object, and the latest action string in your Redux store.

const router = curi(history, routes);
const reducer = combineReducers({
  curi: curiReducer,
  ...
});
const store = createStore(reducer);
syncResponses(store, router);
const { curi } = store.getState();
// curi = { router, response, action }
