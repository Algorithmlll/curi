@curi/redux

v
GitHub RepoNPM Package

About#

This package allows you to synchronize the response objects generated by Curi with your Redux store. Whenever location changes occur, they will be dispatched to your store. If you use this (with a React application), you do not have to use the <Navigator>.

Installation#

You can install the redux package from NPM.

npm install @curi/redux

If you prefer to use script tags, Unpkg will always have the latest version of redux available for you.

<script src="https://unpkg.com/@curi/redux@undefined/dist/curi-redux.js" />

There is also a minimized version available if you change the filename to redux.min.js. The package will be attached to the window as window.CuriRedux.

API#

import {
  syncResponses,
  responseReducer,
  configReducer
} from '@curi/react-block';

syncResponses#

syncResponses is responsible for linking your Redux store with your Curi configuration object. It subscribes to your configuration object with a function that will dispatched a "location changed" event to the Redux store.

Note:

You will probably want to wait for your configuration object to generate its initial response before syncing by using config.ready. Otherwise, the initial response will be undefined.

const config = createConfig(history, routes);
const store = createStore(reducer);

config.ready().then(() => {
  syncResponses(store, config);
});

responseReducer#

You can use the responseReducer to automatically store the latest response object in your Redux store.

const reducer = combineReducers({
  response: responseReducer,
  ...
});
const store = createStore(reducer);

configReducer#

The configReducer can be used to store your Curi configuration object in your Redux store. This isn't really necessary, but can be convenient.

const reducer = combineReducers({
  curi: configReducer,
  ...
});
const store = createStore(reducer);
