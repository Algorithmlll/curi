Side Effects

Explanation#

Side effects are pretty straightforward. Once a response has completed (the route's match.initial and/or match.every functions have resolved), the response's properties are used to create a JavaScript object. Then, any response handler functions are called and passed that JavaScript object . Side effects are basically permanent response handlers (they cannot be removed). Side effects can either be run before (default) or after response handler that were set using router.respond. They receive the new response and an object with information about the navigation.

A side effect function just does something using its arguments. It is a response handler, but unlike ones assigned using router.respond, side effects cannot be removed.

You pass any side effect functions that you want to use to the curi call, using the sideEffects property of the options object.

import curi from 'curi';
import mySideEffect from './mySideEffect';

const router = curi(history, routes, {
  sideEffects: [mySideEffect]
});
