On GitHub

Installation

You can install the curi-addon-prefetch package from NPM.

npm install curi-addon-prefetch

If you prefer to use script tags, Unpkg will always have the latest version of curi-addon-prefetch available for your.

<script src="https://unpkg.com/curi-addon-prefetch@0.2.0/dist/curi-addon-prefetch.js" />

There is also a minimized version available if you change the filename to curi-addon-prefetch.min.js

API

createPrefetchAddon

curi-addon-prefetch has one, default export function (so you can import it with whatever name you want to. It is an addon factory that will add an prefetch function to your configuration object's addon property.

import createConfig from 'curi';
import prefetch from 'curi-addon-prefetch';

const config = createConfig(history, routes, { addons: [prefetch] });

The prefetch addon allows you to call a route's load function manually. Why would you want to do this? Preloading data can give your users a faster navigation time when navigating to a page whose data has already been loaded.

Note: You should only use this if you implement some sort of caching/lookup in your load functions. The load function will be recalled when the user actually navigates to the route, so the benefit comes from the load function using a cached value instead of sending a new request to your server.
// call a route's load function manually
config.addons.prefetch('User', { id: 2 })

This addon will only register routes that have a load function in their load object.
